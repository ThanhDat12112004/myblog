Mối quan hệ giữa các lớp: Kế thừa, Thành phần và Tổng hợp

29 Tháng 12, 2024 8 phút đọc

Trong lập trình hướng đối tượng (OOP), các mối quan hệ giữa các lớp là một khái niệm cốt lõi giúp chúng ta tổ chức và cấu trúc code một cách hiệu quả. Ba mối quan hệ chính cần hiểu là: Kế thừa (Inheritance), Thành phần (Composition), và Tổng hợp (Aggregation).

1. Kế thừa (Inheritance)

Kế thừa là một cơ chế cho phép một lớp mới (lớp con) được xây dựng dựa trên một lớp đã tồn tại (lớp cha). Lớp con sẽ kế thừa các thuộc tính và phương thức từ lớp cha.

Ví dụ về kế thừa:

public class Car {
   private String model;
   private int maxSpeed;
   private int yearOfManufacture;

   public Car(String model, int maxSpeed, int yearOfManufacture) {
      this.model = model;
      this.maxSpeed = maxSpeed;
      this.yearOfManufacture = yearOfManufacture;
   }
}

public class Truck extends Car {
   public Truck(String model, int maxSpeed, int yearOfManufacture) {
      super(model, maxSpeed, yearOfManufacture);
   }
}

Lưu ý: Java không hỗ trợ đa kế thừa - một lớp chỉ có thể kế thừa từ một lớp cha duy nhất.

2. Thành phần (Composition)

Thành phần là mối quan hệ "has-a" chặt chẽ, trong đó một đối tượng là một phần không thể tách rời của đối tượng khác.

Ví dụ về thành phần:

public class Engine {
   private int horsepower;
}

public class Car {
   private Engine engine; // Composition: Car has-a Engine
   public Car() {
      engine = new Engine();
   }
}

3. Tổng hợp (Aggregation)

Tổng hợp là mối quan hệ "has-a" lỏng lẻo hơn, trong đó các đối tượng có thể tồn tại độc lập.

Ví dụ về tổng hợp:

public class Student {
   private String name;
}

public class StudentGroup {
   private List<Student> students; // Aggregation: Group has Students
}

So sánh các mối quan hệ

  • Kế thừa (is-a): Mô tả mối quan hệ "là một". Ví dụ: Truck là một Car.
  • Thành phần (has-a chặt chẽ): Đối tượng con không thể tồn tại độc lập. Ví dụ: Engine trong Car.
  • Tổng hợp (has-a lỏng lẻo): Các đối tượng có thể tồn tại độc lập. Ví dụ: Student trong StudentGroup.
