Đóng Gói Trong Java - OOP

29 Tháng 12, 2024 10 phút đọc

Đóng gói là một trong những nguyên tắc cơ bản của lập trình hướng đối tượng, cho phép bạn kết hợp dữ liệu và các phương thức để làm việc với dữ liệu đó trong một "gói" (capsule).

Khái Niệm Đóng Gói

Trong Java, lớp chính là một capsule, chứa:

  • Dữ liệu (các trường)
  • Các phương thức để làm việc với dữ liệu đó

Ví Dụ Thực Tế

Giống như cách bạn có thể lái xe mà không cần hiểu chi tiết về động cơ đốt trong, trong lập trình, bạn có thể sử dụng các đối tượng mà không cần biết chi tiết triển khai bên trong.

Ví dụ với Vehicle Class

public class Vehicle { public void gas() { /* Some complicated things happen inside a car. As a result, it moves forward */ } public void brake() { /* Some complicated things happen inside a car. As a result, it slows down */ } }

Ẩn Thông Tin (Information Hiding)

Một ví dụ về cách không nên thiết kế class:

public class Cat { public String name; public int age; public int weight; public Cat(String name, int age, int weight) { this.name = name; this.age = age; this.weight = weight; } }

Cách Thực Hiện Đóng Gói

  • Sử dụng access modifiers (private, protected, default)
  • Implement getters và setters
  • Kiểm soát trạng thái đối tượng thông qua các phương thức public

Lợi Ích Của Đóng Gói

  • Giám sát chính xác trạng thái đối tượng, ngăn chặn các giá trị không hợp lệ
  • Cung cấp giao diện người dùng đơn giản và thân thiện
  • Cho phép thay đổi code bên trong mà không ảnh hưởng đến người dùng

Nhớ rằng: Đóng gói không chỉ là việc kết hợp dữ liệu và phương thức, mà còn bao gồm cả việc ẩn thông tin không cần thiết khỏi người dùng, giúp code dễ bảo trì và an toàn hơn.
