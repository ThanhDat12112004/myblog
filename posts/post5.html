Tính Đa Hình Trong Java - OOP

29 Tháng 12, 2024 8 phút đọc

Đa hình là khả năng làm việc với một số loại như thể chúng là cùng một loại. Hơn nữa, hành vi của các đối tượng sẽ khác nhau tùy thuộc vào loại của chúng.

Ví Dụ Về Tính Đa Hình

Hãy xem xét một ví dụ với lớp cha Cat và các lớp con:

public class Cat { public void run() { System.out.println("Run!"); } }

Các lớp con kế thừa từ Cat:

public class Lion extends Cat { @Override public void run() { System.out.println("Lion runs at 80 km/h"); } } public class Tiger extends Cat { @Override public void run() { System.out.println("Tiger runs at 60 km/h"); } } public class Cheetah extends Cat { @Override public void run() { System.out.println("Cheetah runs at up to 120 km/h"); } }

Ứng Dụng Thực Tế

Trong thực tế, tính đa hình giúp chúng ta xử lý nhiều đối tượng khác nhau thông qua một giao diện chung, giảm thiểu độ phức tạp của code.

Ví dụ với lớp Dolittle

public class Dolittle { public void healCat(Cat cat) { System.out.println("The patient is healthy!"); cat.run(); } }

Lợi Ích Của Tính Đa Hình

  • Giảm độ phức tạp của code bằng cách sử dụng một giao diện chung cho nhiều đối tượng
  • Cho phép mở rộng code dễ dàng mà không cần thay đổi logic hiện có
  • Duy trì tính linh hoạt trong việc xử lý các đối tượng khác nhau

Khi 'sự khái quát hóa' này là không mong muốn và thay vào đó chúng ta cần mỗi loài cư xử khác nhau, thì mỗi loại sẽ làm việc riêng của mình. Nhờ tính đa hình, bạn có thể tạo một giao diện duy nhất cho nhiều loại lớp, giúp chương trình ít phức tạp hơn.
